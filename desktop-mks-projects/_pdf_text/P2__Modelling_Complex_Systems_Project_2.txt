1 Problem statement
In this project we are doing numerical experimentation on classical examples
of cellular automata to illustrate and get a deep understanding of patterns in
complex systems and how they can change with update rules. The cellular
automata models we will be exploring are: Conway’s Game of Life (CGL),

2 Problem 1: Conway’s Game of Life
2.1 Problem statement
In this section of the lab, we study Conway’s Game of Life (a cellular au-
tomaton) to implement and simulate the behavior of logical gates: AND, OR,
and NOT. The primary objective is to verify that simple local update rules
applied to a grid of cells can result in universal computation, specifically by
manipulating ”glider” patterns to transmit and process signals. The goal is
to verify that our configurations produce the correct logical outputs for all
possible input combinations.
2.2 Model and parameters
Conway’s Game of Life operates on a two-dimensional orthogonal grid where
each cell is either dead (0) or alive (255). The state of the grid is updated
synchronously in discrete time steps based on the number of living cells in a
3×3 neighborhood:
•A live cell survives if it has 2 or 3 live neighbors.
•A live cell dies if it has fewer than 2 (loneliness) or more than 3 (over-
population) live neighbors.
•A dead cell becomes alive if it has exactly 3 live neighbors (reproduc-
tion).
In our modeling context, binary inputs are physically encoded into the simu-
lation by the presence or absence of ”stopper” blocks. If an input parameter
2

===== PAGE 4 =====
is 0, a stopper block is spawned in the trajectory of the signal glider, anni-
hilating it prematurely. If the input parameter is 1, the stopper is removed,
allowing the glider to traverse the grid and collide with other patterns. The
output is read by visually observing the final state of the grid; an output of
1 corresponds to a glider successfully escaping the collision zone, while an
output of 0 corresponds to total mutual annihilation of the gliders.
2.3 Logical gates in the Game of Life
We define what the gates (AND, OR, and NOT) that we want to realize in
CGL mean logically in our case.
Gate Inputs Output rule Meaning
AND A, B A∧B both true
(00→0,01→0,10→0,11→1)
OR A, B A∨B at least one true
(00→0,01→1,10→1,11→1)

Here, the AND gate is realized by having two gliders A and B interact in the
same input region allowing one glider to reach the output region. The OR
gate allows a glider to appear in the output when at least one of the input
signals are present and the NOT gate is realized by having a reference glider
interact with an input glider allowing an output glider to appear only if the
input signal is missing.

3 Problem 2: Neuron Cellular Automaton
3.1 Problem statement
In this section, we study an excitable medium modeled by a simple three-
state cellular automaton that mimics the firing cycle of biological neurons.
The goal is to investigate how local deterministic rules governing excitation
and recovery can lead to complex macroscopic behavior. Specifically, we
aim to design initial configurations that produce three distinct dynamical
phenomena: a stable moving pattern, a moving pattern that spawns new
structures along its path, and a system exhibiting strictly periodic motion.
3.2 Model and parameters
The automaton operates on anN×Ngrid with periodic boundary conditions,
effectively forming a torus. Each cell (representing a neuron) can exist in one
of three states:
•Ready (0):The resting potential state, waiting for a stimulus.
•Firing (1):The action potential state, transmitting the signal.
•Resting (2):The refractory period, unable to fire regardless of stim-
ulus.
10

===== PAGE 12 =====
The update rules applied synchronously to all cells at each time step are as
follows:
1. A neuron in thereadystate becomesfiringin the next step if exactly
two of its eight surrounding neighbors (Moore neighborhood) are cur-
rently firing.
2. A neuron in thefiringstate automatically transitions to theresting
state in the next step.
3. A neuron in therestingstate automatically recovers to thereadystate
in the next step.
3.3 Design of Initial Configurations
To satisfy the three specific behavioral requirements of the assignment, tar-
geted initial configurations were designed by manipulating the placement of
FIRING (1) and RESTING (2) cells on a background of READY (0) cells.
Task A: Stable Moving PatternTo achieve a stable pattern that
moves forward in time, a minimal 2-cell wide ”glider” was constructed. The
front of the glider consists of two FIRING cells, immediately followed by a
tail of two RESTING cells. Because the front is exactly two cells wide, the
READY cells directly ahead of the front perceive exactly two firing neighbors.
This perfectly satisfies the excitation rule, causing the glider to pull itself
forward in a straight line infinitely.

1def get_next_state(self):
2"""Vectorized update"""
3new_grid = np.zeros_like(self.grid)
4
5# Identify Excited Bit/boolian-Mask (1 <= state <= e)
6is_excited = (self.grid >= 1) & (self.grid <= self.e)
7
8# Von Neumann Neighborhood check
9# Shifted grids: neighbors outside are 0 (not excited) due to padding
10up = np.pad(is_excited, ((0,1),(0,0)), mode=’wrap’)[1:, :]
11down = np.pad(is_excited, ((1,0),(0,0)), mode=’wrap’)[:-1, :]
12left = np.pad(is_excited, ((0,0),(0,1)), mode=’wrap’)[:, 1:]
13right = np.pad(is_excited, ((0,0),(1,0)), mode=’wrap’)[:, :-1]
14
15neighbor_excited = up | down | left | right
16
17# Rule 1: State 0 transitions
18mask_0 = (self.grid == 0)
19new_grid[mask_0 & neighbor_excited] = 1
20new_grid[mask_0 & ~neighbor_excited] = 0
21
22# Rule 2: 1 <= s <= N-2 transitions
23mask_advance = (self.grid >= 1) & (self.grid <= self.N - 2)
24new_grid[mask_advance] = self.grid[mask_advance] + 1
25
26# Rule 3: s = N-1 transitions to 0
27mask_reset = (self.grid == self.N - 1)
28new_grid[mask_reset] = 0
29
30return new_grid

4.8 Interpretation
It is generally the case that the automatons, unless given certain patterns as
initial conditions, evolve into complicated cyclic patterns, where the period
matches the models state parameter. An exception to this state parameter
period matching, usually occur if the model has degenerate parameters or
attains a degenerate state e.g. the entire grid being constantly zero.
4.9 Conclusion
The GreenBergHasting cellular automaton has a propensity to settle into
orbits directly linked to it’s parameters, mainly it’s state parameter. This is
however of no surprise, as the most basic cycle a grid square can traverse is
simply cycling it’s states. An exception to this is wave propagation, where
the statek= 0 part of the wave might be thicker, delaying the period by
one cycle.
